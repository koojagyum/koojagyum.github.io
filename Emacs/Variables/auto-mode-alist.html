auto-mode-alist

auto-mode-alist는 files.el 에 정의되어 있다.

이 변수를 어떤 파일에서 로컬 변수로 사용하는 것은 위험할 수도 있으니 주의한다.

파일 이름을 나타내는 패턴과 그에 대응하는 매이저 모드 함수들로 이루어진 alist(Associsation List) 이다. 각각의 엘리먼트는 (REGEXP . FUNCTION) 혹은 (REGEXP FUNCTION NON-NIL) 이런 식으로 이루어져 있다. (NON-NIL은 nil값이 아닌 다른 어떤 것을 나타낸다. 아무 값이나 상관 없다.) REGEXP에 매칭되는 어떤 파일을 열면 해당되는 FUNCTION 에서 지정하는 모드를 사용하는 것이다. 즉 nil이 아닌 FUNCTION은 호출된다.

만일 어떤 엘리먼트가 (REGEXP FUNCTION NON-NIL) 형태를 갖는다면 FUNCTION이 불린 다음에 REGEXP와 매칭되는 접미사1를 삭제하고 다른 매칭을 찾아 리스르를 다시 뒤진다2.

파일에 대응하는 FUNCTION이 `archive-mode'3라면 `auto-coding-alist'4에도 `no-conversion' coding system 으로 되어 있어야 한다.

`interpreter-mode-alist'에 대한 문서도 찾아보기 바란다. 이것은 파일을 열었을 때 첫 라인에 명시한 인터프리터 이름을 보고 매이저 모드를 활성화 시키는 것에 대한 변수이다. 그리고 `magic-mode-alist'도 역시 비슷한 형태로서 파일 안의 내용과 매칭시켜서5 매이저 모드를 결정한다.
