OpenGL wiki의 OpenGL Object 문서 번역.

OpenGL Object 는 몇 가지 상태값을 갖는 OpenGL의 구조체라고 보면 될 것 같다. 이것들이 컨텍스트에 바인딩되면 이들이 가진 상태값이 컨텍스트의 상태값으로 매핑된다. 그러므로 현재 컨텍스트의 상태값이 변화하면 이 객체에 반영되며 컨텍스트 상태에 영향을 받는 함수들도 이 객체의 상태값을 사용한다.

OpenGL은 "상태 머신" 이다. API 호출은 OpenGL의 상태를 변화시키고 어떤 상태값을 쿼리하여 가져오기도 하며, OpenGL이 무엇인가 렌더링할 때에는 현재 상태를 참고하기도 한다.

OpenGL Object 들은 모두 상태값을 저장하는 컨테이너이다. 각각의 타입의 Object는 자신이 품고 있는 특정 상태에 의해 정의된다. 어떤 OpenGL Object는 특정 상태들의 그룹을 캡슐화하는 방식으로 사용되기도 하며 한 번의 함수 호출로 상태값들을 모두 바꾸어버리기도 한다.

Remember: 이 문서는 OpenGL이 그 스펙을 어떻게 정의하고 있는냐에 대한 것이다. 실제 드라이버에서 이런 Object가 어떻게 구현되었는가는 또 다른 문제이다. 하지만 이건 걱정할 문제는 아니고 중요한 것은 Object와 상태값이 어떻게 스펙에 정의된 대로 상호 작용하냐에 대한 것이다.

1 Object Creation and Destruction

Object를 생성하기 위해서는 정수 형태의 Object의 이름을 생성해야 한다. 이렇게 생성되는 정수값은 해당 Object에 대한 참조를 나타낸다. Object가 컨텍스트에 바인드되면 처음에는 디폴트 상태값을 갖게 된다.

Object의 이름을 생성하는 함수는 glGen* 의 형태를 이룬다(*은 Object 타입의 복수형). 이런 형태의 함수들은 모두 같은 프로토타입이다. 

void glGen*(GLsizei n, GLuint *objects);

위 함수는 주어진 타입으로 n개의 Object를 생성하여 objects 파라미터에 배열로 저장한다. 이렇게 하여 한 번의 호출로 여러 개의 Object를 생성할 수 있다.

Object의 이름은 언제나 GLuint 이다. 이름은 포인터가 아니다. 이름은 레퍼런스이며 그 Object에 대한 식별자이다. 이름은 0이 아닌 32비트의 unsigned integer이다. 0인 Object는 특별한 상황에 대해 내정된 객체이다. 자세한것은 이후의 내용을 참고하자.

Legacy Note: OpenGL 3.0 이전에서는 사용자가 generation 단계를 완전히 무시하는게 가능했다. 예를 들어 사용자는 "3"이 유효한 Object 이름이라고 보고 이를 Object처럼 사용할 수 있었다. 실제 구현은 이런 식의 접근을 허용하고 "3번" Object를 처음으로 사용할 때 Object를 몰래 생성해준다. GL 3.0 에서 이러한 방식은 사용하지 않도록 권장되었다. GL 3.1 이후부터는 이런 방식이 아예 허용되지 않고 있다. 아무튼 OpenGL 버전과 관계 없이 glGen* 을 통해 Object를 만드는 것이 임의로 번호를 따서 만드는 것보다 더 좋은 연습이 될 것이다.

Object를 다 사용하였다면 지워야 한다. 삭제하는 함수는 glDelete* 의 형태를 이룬다(*은 Object 타입이다). 삭제 함수들은 아래와 같은 프로토타입이다. 

void glDelete*(GLsizei n, const GLuint *objects);

삭제 함수 역시 glGen* 함수처럼 동작한다. 단, 복수의 Object를 생성하는게 아니라 지우는 것이다. 유효하지 않은 Object 값이나 0에 대해서는 무시된다.

OpenGL Object가 지워지면 이들의 이름은 더 이상 유효하지 않은 것으로 간주된다. 이렇게 삭제하고 난 다음에 바로 glGen* 을 사용하면 방금 전에 지웠던 그 이름은 다시 사용될 수도 있고 아닐 수도 있다. 그래서 이에 대해 미리 예상하고 코드를 짜서는 안된다.

1.1 Deletion unbinding

Object가 현재 컨텍스트에 바인드된 상태(현재 컨텍스트 에 대해서만으로 한다)로 지워졌다면 그 Object는 그 컨텍스트에 대해 언바인드될 것이다.

Note: "attachment" 가 아니라 "binding" 이다. 바인딩은 컨텍스트에 대한 것이다. attachment는 어떤 Object가 다른 것을 참조하게 되었을 때 무엇을 하느냐에 대한 것이다. attachment는 위의 상황에 대해 동일하게 동작하지 않는다.

어떤 Object들은 비정상적인 방식으로 컨텍스트와 연관되어 있다. 이런 방법이란 아래 내용을 포함한다.

  • glBindBufferRange 혹은 이와 비슷한 형태의 함수에 의해 인덱스가 지정된 타겟에 바인드된 Buffer Object.
  • Image 로 바인드된 Texture.

위의 경우 Object가 삭제되더라도 바인딩이 해제되지 않는다. 해제가 가능한 바인딩은 해당 Object 타입의 glBind* 함수를 사용해서 바인딩이 해제되는 유형의 바인딩이어야 한다.

따라서 glBindBufferRange(GL_UNIFORM_BUFFER, …)를 사용하여 바인드된 버퍼를 삭제하는 것은 마치 glBindBuffer(GL_UNIFORM_BUFFER, 0)을 호출한 것과 같은 것이다. 이렇게 하면 인덱스가 지정된 타겟에서 바인드를 해제하지 않는다.
