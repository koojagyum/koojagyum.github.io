.gitignore dosen't work!

.gitignore 파일이 정상적으로 전혀 동작하지 않는 경우가 있는데 untrack 할 파일이 이전에 이미 저장소에 들어가서 지금까지 사용하고 있던 것이라면 이러한 현상이 발생할 수 있다. .gitignore를 쓴다는 것은 해당 파일을 실제로 삭제하고 싶지 않은 경우이기 때문에 git 시스템에서만 삭제해 주어야 한다. 이 경우 git rm --cached filename 이런 식으로 해당 파일을 시스템에서 지워준다.

궂이 filename을 주기 귀찮은 경우 아래와 같이 하면 편하다.

  1. 우선 .gitignore 를 작성해 주고
  2. 아래 명령어를 실행하여 index(staging area)에서 변경된 이력을 지워준다.

    git rm -r --cached .

  3. 그 다음 수정 사항을 다시 staging 하고

    git add .

  4. 마지막으로 커밋한다.

    ​git commit -m ".gitignore is now working"​
