org-publish-project-alist

The variable org-publish-project-alist 번역.

퍼블리싱 관련해서 대부분의 설정은 org-publish-project-alist 라는 변수 하나로 가능하다. 이 리스트 안의 각각의 엘리먼트는 하나의 프로젝트를 나타내며 아래 예제에서 둘 중 하나의 형태가 된다. 

("project-name" :property value :property value ...)
; i.e., a well-formed property list with alternating keys and values
or 
("project-name" :components ("project-name" "project-name" ...))

두 경우 모두 property와 value를 지정하여 프로젝트를 설정하고 있다. 여기서 프로젝트는 어떤 파일들이 퍼블리싱될 것인가와 함께 이들을 퍼블리싱하면서 사용할 설정값들을 나타낸다. 어떤 프로젝트가 위 예제의 두 번째 형태로 표현되었다면 :compoentns 프로터티의 각 멤버들은 서브 프로젝트가 되고, 이들은 서로 다른 퍼블리싱 옵션들과 소스 파일들을 정의하게 될 것이다. 이렇게 "메타 프로젝트"를 하나 퍼블리싱하게 되면 그 아래 컴포넌트들이 순차적으로 퍼블리싱된다.
